# Lab Guide — Azure DevOps Foundation Setup

## Lab objective
The objective of this lab is to prepare a clean and reliable Azure DevOps learning environment that can support all later modules in this repository.

This lab focuses on platform readiness, local environment readiness, and workflow clarity. The learner will create the initial Azure DevOps structure, confirm access, prepare the machine for future labs, and validate that the setup is strong enough for the rest of the course.

## What you will build or configure
In this lab, you will establish the base working environment for the repository. This includes Azure DevOps access, organization and project readiness, local machine preparation, and tool verification.

This lab does not aim to build a large solution. Its purpose is to make sure the learner is starting from a clean, stable, and repeatable foundation.

## Why this lab matters
Many learners try to start directly with repos, pipelines, or Kubernetes topics without first stabilizing the environment. That usually creates unnecessary confusion, tool errors, and access problems later.

This lab removes that friction early and creates a disciplined starting point for the entire course.

## Expected final outcome
By the end of this lab, the learner should have:
- working Azure DevOps access
- a clean Azure DevOps organization and project structure
- a prepared local environment with key tools available
- confidence that later modules can be started without setup-related confusion

## Skills gained
- Understanding the role of Azure DevOps in modern delivery workflows
- Preparing a local environment for DevOps-focused labs
- Creating a clean Azure DevOps starting structure
- Verifying environment readiness before beginning real implementation modules

## Real-world relevance
In real projects, successful delivery starts with platform readiness and environment discipline. Teams lose time when engineers begin work without proper access, tool readiness, naming discipline, or platform understanding.

This lab reflects the same real-world need: prepare first, then execute with clarity.

## Completion standard
A learner should not mark this lab complete until:
- Azure DevOps access is confirmed
- the working project structure is created
- local tools are available and usable
- readiness has been verified through validation checks
- cleanup guidance has been reviewed
