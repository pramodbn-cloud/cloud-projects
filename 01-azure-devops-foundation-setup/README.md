# 01 — Azure DevOps Foundation Setup
![Week-1 Plan](../Posts/1.png)

## Overview
This module establishes the starting point for the entire learning repository. It introduces Azure DevOps as a delivery platform and helps the learner create the basic environment required to continue with the rest of the course in a structured way.

This is not just an account-creation step. This module is where the learner builds the initial platform foundation, local machine readiness, and working discipline that all later modules will depend on.

## Why this module matters
A strong Azure DevOps journey cannot begin with pipelines or deployments alone. It begins with a properly prepared platform, a clean local setup, and a clear understanding of what Azure DevOps is meant to do in a real engineering workflow.

If the foundation is weak, later modules become confusing because learners start troubleshooting setup problems instead of learning actual delivery concepts. This module prevents that by creating a clean starting point.

## What you will learn
- What Azure DevOps is and why it is used in engineering workflows
- How Azure DevOps organization and project structure works at a high level
- How to prepare the local machine for DevOps-focused work
- How to verify readiness before moving into planning, repos, and pipelines

## Workflow position
This is the first module in the repository, so it does not depend on earlier folders. It prepares the learner for everything that follows, especially work planning, repositories, source control workflow, and CI/CD foundations.

Without this module, the learner may understand concepts in theory but still struggle to execute the later hands-on sections with confidence.

## Included in this folder
- Module overview
- Post image
- Hands-on lab
- Validation guide
- Troubleshooting guide
- Cleanup guide

## Expected outcome
By the end of this module, the learner should be able to:
- explain what Azure DevOps is in simple, practical terms
- create or access an Azure DevOps organization and project
- verify that the local environment is ready for future modules
- understand the starting structure required for the rest of the course

## Recommended approach
1. Read this overview fully  
2. Review the post image inside `post-assets/`  
3. Complete the lab files in order  
4. Confirm readiness using the validation guide  
5. Do not proceed to the next module until your environment is clean and working  

## Next module
The next module is **Agile Planning + Traceability**.

That module builds directly on this setup by showing how teams use Azure DevOps not only as a code platform, but as a planning and work-management platform. Once the learner has a working environment, they are ready to understand how work is structured, tracked, and connected to delivery.
