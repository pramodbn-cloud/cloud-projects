# Lab Guide — Instrumentation + Automation + Capstone

## Lab objective
The objective of this lab is to help the learner create a simple but meaningful operational view of the Azure DevOps workflow built in the earlier modules.

This lab focuses on dashboards, widgets, notifications, event-driven automation concepts, and full lifecycle reflection. The goal is not to create a heavy enterprise observability platform, but to make the learner comfortable with how Azure DevOps supports visibility, reaction, and coordination.

## What you will build or configure
In this lab, you will create or review a simple dashboard-driven monitoring view for your project, identify useful signals from earlier modules, review or configure notifications, and introduce an event-driven automation pattern using service hooks or webhooks conceptually or practically.

You will also use this lab as a capstone to connect the full foundation-phase journey into one coherent DevOps workflow.

## Why this lab matters
Delivery systems need more than execution. They need visibility and reaction.

This lab matters because planning, repositories, builds, releases, packages, and security all become more useful when teams can observe them clearly and respond to events faster. A workflow that cannot be seen and understood is difficult to improve.

## Expected final outcome
By the end of this lab, the learner should have:
- a simple dashboard or visibility layer for the project
- an understanding of useful Azure DevOps widgets and signals
- awareness of notification and service-hook automation patterns
- a full foundation-phase capstone understanding
- confidence in explaining the complete Azure DevOps lifecycle

## Skills gained
- Thinking in terms of operational visibility rather than only execution
- Building a dashboard-oriented status view
- Understanding notification and service-hook automation patterns
- Explaining the full Azure DevOps foundation flow end to end

## Real-world relevance
In real engineering teams, dashboards, alerts, notifications, and integrations help teams monitor status, spot problems early, and coordinate action across tools and services. This module reflects that same shift from isolated activity to visible, managed delivery.

## Completion standard
A learner should not mark this lab complete until:
- the dashboard or visibility story is clear
- the learner understands which signals matter and why
- notification and automation patterns are understood
- the full foundation workflow can be explained clearly
- validation confirms that the capstone understanding is strong
