# Folder 08 — Instrumentation + Automation + Capstone
![Week-9 Plan](../Posts/9.png)

## Overview
This module brings together the work completed in the earlier foundation modules and turns it into an observable, operationally meaningful delivery system. It introduces the learner to instrumentation and automation patterns inside Azure DevOps so that work, code, builds, releases, and operational events can be seen, tracked, and acted on more effectively.

This is the capstone module for the Azure DevOps foundation phase. It helps the learner move from “I can use the platform” to “I can observe and coordinate the platform in a structured way.”

## Why this module matters
A delivery workflow is not mature simply because it works. It becomes mature when teams can observe its health, understand its trends, react to failures, and automate the right follow-up actions.

Without instrumentation, teams lose visibility into progress, pipeline health, test behavior, and release outcomes. Without automation hooks and event-driven response patterns, teams depend too heavily on manual follow-up and slow reaction cycles.

This module builds the visibility-and-response mindset that supports reliable DevOps operations.

## What you will learn
- How dashboards and widgets improve visibility into work and delivery health
- Why notifications and event-driven automation matter in DevOps workflows
- How service hooks and webhook-style integrations support external actions
- How to connect planning, repos, pipelines, packages, security, and visibility into one foundation-level delivery story

## Workflow position
This module builds on the full foundation phase:
- planning and traceability
- source control
- CI
- CD
- package management
- security and compliance

Now the learner uses those foundations to build an operational view of the system and understand how Azure DevOps becomes an observable and automation-ready platform.
