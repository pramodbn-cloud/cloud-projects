# Cleanup

## Purpose of this file
This file defines how to leave the Agile Planning + Traceability lab in a clean and reusable state.

This module does not usually create billable cloud resources, but it can create clutter if many poor-quality or duplicate work items are left behind. Clean planning is important because later modules may build on this project structure.

## What should be removed
Remove or clean up:
- duplicate or accidental work items
- test items that do not fit the scenario
- confusing naming experiments
- extra iteration entries created only for trial and no longer needed

If something adds noise without teaching value, it should be removed.

## What can be retained
The following should usually be kept:
- the main planning scenario
- the clean work item hierarchy
- the useful iteration or sprint setup
- the query if it helps later review
- notes on how the hierarchy was designed

These will help in later modules when connecting planning to repositories and delivery workflows.

## Cleanup sequence

1. Review the full work item list  
2. Keep only the items that clearly support the chosen scenario  
3. Remove duplicates or badly named trial items  
4. Keep one clean hierarchy that can still be understood later  
5. Review sprint or iteration entries and remove unnecessary clutter if applicable  
6. Confirm that the project now contains a neat and reusable planning model  

## Cost awareness note
This module does not mainly affect cloud cost. Its main risk is planning clutter.

Poor cleanup here can reduce the quality of later learning because the learner may forget which work items were intentional and which were only testing artifacts.

## Post-cleanup validation
After cleanup:
- one clean scenario should remain
- the hierarchy should still make sense
- the board or backlog should be readable
- the learner should feel ready to connect this planning structure to source control in the next module

## Final note
This lab is fully complete only when:
- the planning structure is created
- validation is complete
- traceability is understood
- unnecessary planning clutter has been removed
- the project is clean enough for Folder 03
