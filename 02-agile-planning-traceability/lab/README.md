# Lab Guide — Agile Planning + Traceability

## Lab objective
The objective of this lab is to help the learner set up and understand a clean Azure Boards planning workflow inside the Azure DevOps project created earlier.

This lab focuses on work structuring, sprint readiness, hierarchy understanding, and traceability thinking. The learner will create work items, organize them into a meaningful structure, review sprint alignment, and understand how planning becomes the foundation for technical delivery.

## What you will build or configure
In this lab, you will create a simple but realistic planning model inside Azure DevOps using work items and iteration structure.

You will define a small delivery scenario, create its work hierarchy, organize it inside Azure Boards, and review how the team would track progress from high-level objective down to execution-level activity.

## Why this lab matters
A strong engineering workflow begins with clarity of work. When planning is weak, everything that follows becomes noisy: branch names feel disconnected, pull requests lose business meaning, pipeline runs are hard to relate back to outcomes, and progress reporting becomes unreliable.

This lab helps the learner build the thinking discipline that supports clean delivery later.

## Expected final outcome
By the end of this lab, the learner should have:
- a structured Azure Boards work hierarchy
- a basic sprint or iteration setup
- a working understanding of agile planning flow inside Azure DevOps
- the ability to explain how traceability connects planning to execution

## Skills gained
- Creating and organizing work items in Azure Boards
- Understanding backlog hierarchy and delivery breakdown
- Building a simple sprint-ready planning structure
- Thinking in terms of traceability rather than isolated tasks

## Real-world relevance
In real teams, successful delivery depends on how well work is shaped before execution begins. Engineers, leads, and managers rely on planning systems not just to assign work, but to preserve context, track ownership, measure progress, and report outcomes.

This lab reflects that same real-world planning discipline.
