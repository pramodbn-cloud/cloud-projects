# Folder 02 — Agile Planning + Traceability
![Week-2 Plan](../Posts/2.png)

## Overview
This module introduces Azure DevOps as a work planning and execution tracking platform. It helps the learner understand how delivery work is structured before code is written, how work items are connected to business intent, and how teams create visibility from requirement to execution.

This is where Azure DevOps starts becoming more than a tool collection. The learner begins to see how work flows through a team in an organized, trackable, and measurable way.

## Why this module matters
Many learners first see Azure DevOps only through repositories and pipelines, but real engineering delivery starts before code is committed. Teams need a system to capture ideas, break them into deliverable units, prioritize effort, assign ownership, and track movement across iterations.

Without planning and traceability, delivery becomes disconnected. Code changes lose business context, progress becomes harder to measure, and teams struggle to explain what was done, why it mattered, and how it moved through the workflow.

This module builds that missing layer.

## What you will learn
- How Azure Boards helps teams plan, organize, and track delivery work
- The difference between epics, features, user stories, and tasks
- How sprint and iteration structure supports delivery rhythm
- Why traceability matters from business need to implementation

## Workflow position
This module builds directly on Folder 01, where the learner created the Azure DevOps environment. Now that the platform foundation is ready, the next logical step is to understand how work is represented and managed inside that environment.

This module also prepares the learner for the source control and CI/CD modules that follow, because later those activities will need to connect back to planned work.

By the end of this module, the learner should be able to:
